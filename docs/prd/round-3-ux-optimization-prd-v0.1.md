# 第三轮 PRD v0.1（用户体验优化）

## 1. 文档信息

- 产品名称：Yunwu 图片工作台
- 文档版本：v0.1
- 文档日期：2026-04-30
- 文档性质：第三轮开发基准稿
- 基线版本：`v0.3.0`

## 2. 本轮结论

第三轮目标是优化个人用户体验，但不得破坏第二轮已经通过的核心创作闭环。

本轮最高优先级不是堆新页面，而是守住以下主链路：

- 创建页提交后正确送达所选模型。
- 创建成功后直接进入对应工作台。
- 工作台就是会话管理，不新增独立会话管理页作为主路径。
- 普通用户页面不暴露研发阶段、系统诊断、provider 内部概念。
- 每个功能实现后立即测试，不能集中到最后才发现主链路损坏。

## 3. 产品边界

### 3.1 工作台定义

`/workspace/:conversationId` 是会话管理本身。

工作台必须承载：

- 会话列表与当前会话切换
- 当前会话的任务推进
- 状态刷新与失败恢复
- 结果查看
- 再编辑
- 生成变体
- Fork
- 来源追溯

不得新增独立 `/conversations` 或“会话管理标签页”作为普通用户主路径。

### 3.2 创建页定义

`/create` 只负责发起一次创作。

创建页必须满足：

- 文生图可提交
- 上传图编辑可提交
- 来源任务继续创作可预填
- 提交成功后进入 `/workspace/:conversationId`
- 提交失败时保留输入并展示用户可理解原因

### 3.3 配置页定义

配置页服务普通用户当前阶段的自由配置，不承担管理员诊断职责。

配置页必须支持：

- Yunwu `base_url` 切换
- 可用模型列表管理
- 主题切换：明亮、暗色、自选配色
- UI 参数调整与恢复默认

## 4. 默认模型策略

默认向普通用户开放且进入创建页模型列表的模型仅限：

- `gpt-image-2`
- `gemini-3-pro-image-preview`
- `gemini-3.1-flash-image-preview`
- `grok-4.2-image`
- `grok-imagine-image-pro`

`test_provider.txt` 中其他模型默认关闭。用户可在配置页按厂商或模型类型手动启用。

模型接入必须对照 `yunwu-image-api.md`。不同模型族可能使用不同接口路径、请求体、任务查询方式或文件上传方式，不允许用一个临时兼容层把所有模型强行塞进同一 OpenAI Images 请求。

若某模型本轮尚未完整支持，系统应：

- 不在普通创建列表中默认展示
- 在配置页标注能力状态
- 提交时返回清晰原因
- 不生成误导性的占位结果

## 5. base_url 策略

支持两个 Yunwu 访问地址：

- `https://yunwu.ai`
- `https://api3.wlai.vip`

默认使用 `https://yunwu.ai`。用户可主动切换到 `https://api3.wlai.vip`。

切换后应影响后续 provider 调用，并在配置页可见当前选择。

## 6. 本轮需求分级

### 6.1 P0

- 保护创建、编辑、变体、Fork 主链路。
- 创建后直接进入工作台。
- 工作台即会话管理。
- 默认模型收敛为 5 个。
- 模型调用不得错误退化为无意义图片。
- 模型选择 UI 必须可读。
- 普通用户页面移除内部研发文案。

### 6.2 P1

- 配置页支持 base_url、模型启用、主题、UI 参数。
- 首页失败恢复栏简化，只展示最新几个，支持删除或忽略。
- 最近会话、最近任务、最近作品只展示最新几个，支持展开。
- 历史页改为高密度列表，显示最后一次生成图片的小预览图。
- 作品和会话支持归档、软删除、单层分组。

### 6.3 P2

- 为 LLM 接入预留接口，用于未来自动命名会话、优化提示词。
- 更完整的多模型族异步任务状态映射。
- 更细粒度的模型能力标签与风险提示。

## 7. 即时测试门槛

每个功能实现后必须立即执行对应测试。

最低自动化门槛：

- `pnpm local:test`
- `pnpm local:test:smoke`
- 涉及前端页面时执行 `pnpm --filter @yunwu/web lint`
- 涉及后端时执行 `pnpm --filter @yunwu/api lint`

最低真实浏览器门槛：

- 登录普通用户
- 打开 `/create`
- 使用默认模型提交文生图
- 提交后进入 `/workspace/:conversationId`
- 上传图片后发起编辑
- 从已有作品或任务再编辑
- 打开配置页，确认模型列表、base_url、主题控件可读且不溢出
- 打开首页、历史页、作品库，确认列表不会无限下拉堆积

## 8. 用户侧验收标准

本轮交付给用户测试前，必须满足：

- 不再出现 `Round 3` 等内部阶段文案。
- 创建页提交不会返回与提示词无关的占位图或错误伪成功。
- 默认模型列表只出现 5 个模型。
- 配置页文字不越界，明亮和暗色主题均可读。
- 工作台中能完成会话切换、结果查看、继续创作和失败恢复。
- 首页、历史页、作品库在任务较多时仍保持可扫描。
