# v0.5.1 Release Notes

## Summary

v0.5.1 is a small provider hotfix for the Grok Image editing path. It aligns the backend multipart request with the Yunwu Apifox documentation for the Grok Image series while preserving the existing OpenAI Images and Gemini routes.

## Fixed

- Filters Grok Image edit multipart params to documented fields: `aspect_ratio`, `response_format`, `resolution`, `quality`, and `n`.
- Maps the app's shared `size` selector to Grok's documented `aspect_ratio` values, for example `720x1280` to `9:16`.
- Sends only one input `image` for Grok Image editing, matching the documented one-image limit.
- Keeps GPT image edit multipart behavior and Gemini multimodal chat edit behavior unchanged.
